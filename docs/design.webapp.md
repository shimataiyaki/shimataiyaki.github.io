# Webアプリ用 デザイン系譜

「Webサイト用」から骨格を継承しつつ、**アプリ固有のモチーフ（和風・遊び・温もり）を尊重する**ためのデザイン言語。

---

## 基本原則（Webサイト用からの継承）

| 項目 | ルール |
| :--- | :--- |
| **形状** | 角丸ゼロ（`border-radius: 0`）を基本とする。ただし**アプリ固有モチーフ（絵馬・おみくじ札など）は例外** |
| **影** | 原則使わない。ただし立体感が必要なモチーフは控えめに許可（`box-shadow: 4px 4px 0 rgba(0,0,0,0.1)` など） |
| **色彩** | 背景 `#ffffff` / テキスト `#1f2937` / 境界線 `#e2e8f0` / 補足テキスト `#64748b` |
| **識別子** | ヘッダー下部に `3px solid` のアクセントライン |

---

## アクセントカラー

アプリごとに **単一のアクセントカラー** を定義し、以下に適用する。

- ヘッダー下線
- ハンバーガーボタンの線
- リンクホバー時の下線
- 見出しの下線／左ボーダー

| アプリ名 | アクセントカラー | 備考 |
| :--- | :--- | :--- |
| `binary-omikuji` | `#ad241f`（朱赤） | 和風・神社のイメージ |
| `digital-ema` | `#ad241f`（朱赤） | 同上（統一） |
| （今後追加） | アプリのテーマに応じて変更 | |

## 全体レイアウト

```
┌────────────────────────────────────────────┐
│  固定ヘッダー（全幅）                       │
│  [アプリ名]                    [≡]        │
│  （下部に 3px アクセントライン）            │
├────────────────────────────────────────────┤
│                                            │
│         アプリ固有のメインコンテンツ         │
│         （中央1カラム・背景白）             │
│                                            │
├────────────────────────────────────────────┤
│  フッター（線のみ・コピーライト中央）        │
└────────────────────────────────────────────┘
```

- **ヘッダー**：`position: fixed`、常時ハンバーガーメニュー（全幅展開）。
- **メイン**：`max-width: 1200px〜1800px` で中央寄せ。背景白。
- **フッター**：上辺に `1px solid var(--border-color)`。コピーライトのみ。

## ヘッダー & ナビゲーション（共通）

### 構造

```html
<header class="site-header">
  <div class="header-container">
    <a href="#" class="site-title">アプリ名</a>
    <button class="menu-toggle" id="menuToggle">≡</button>
    <ul class="nav-menu" id="navMenu">
      <div class="nav-menu-inner">
        <!-- アプリ固有の最小限ナビ -->
      </div>
    </ul>
  </div>
</header>
```

### 特徴

- **ハンバーガーボタン**：常時表示。`active` クラスで「×」に変化。
- **ナビメニュー**：全幅展開。内部は `.nav-menu-inner` で最大幅に制限。
- **メニュー項目**：アプリ内の主要画面（例：アーカイブ）＋外部リンク（note / GitHub）のみに絞る。

## アプリ固有モチーフの扱い

「Webサイト用」の「角丸ゼロ・影ゼロ」ルールは**構造要素（ヘッダー・フッター・ボタン・通知バナーなど）**に適用する。  
**アプリの核となるモチーフ（おみくじ札、絵馬カードなど）**は、以下の条件を満たす限り、従来の形状を維持してよい。

### 維持してよい要素
- 角丸（`border-radius`）
- 控えめな影（`box-shadow`）
- モチーフ固有の装飾（絵馬の上部金具 `::before` など）

### 維持すべきでない要素
- 過剰なグラデーション
- 複数色の乱立（アクセントカラー＋モチーフ固有の2〜3色に絞る）
- 通知バナーやボタンへの装飾過多

### 色の絞り方（例：digital-ema）

```css
:root {
  --accent-color: #ad241f;      /* アクセント（朱赤） */
  --card-border: #b87c4b;       /* 絵馬の枠（和風茶） */
  --card-bg: #fffaf0;           /* 和紙風の白 */
  --card-shadow: #d9c6b0;       /* 影（控えめなベージュ） */
}
```

---

## 通知バナー（共通パターン）

**背景色・左ボーダーは使わない。**  
フッターと同じく、**下部に細い線のみ**で区切る。

```css
.notification-banner {
  background: transparent;
  border-bottom: 1px solid var(--border-color);
  padding: 0.75rem 1.5rem;
  color: var(--text-color);
}
```

## レスポンシブ

| :--- | :--- |
| 全幅 | ハンバーガーメニューは常に全幅展開 |

## ファイル構成（GASプロジェクト想定）

```
.
├── index.html        # メイン画面
├── archive.html      # アーカイブ画面（必要な場合）
└── コード.gs          # バックエンド（変更不要）
```

- CSSは各HTMLに `<style>` で内包（GASの制約に合わせる）。
- JavaScriptも各HTMLに内包。`google.script.run` によるサーバー連携はそのまま。

---

## 新規アプリへの展開手順

1. **HTMLテンプレートをコピー**（`index.html` など）。
2. `:root` の `--accent-color` をアプリのテーマカラーに変更。
3. ヘッダーの `.site-title` とメニュー項目を書き換え。
4. メインコンテンツ部分にアプリ固有のUIを実装。
5. **モチーフがある場合**、その部分だけ角丸・影を許容するCSSを追加。
